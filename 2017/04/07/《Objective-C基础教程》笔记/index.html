《Objective-C基础教程》笔记

2017-04-07 
1. #import的作用
    #import是由Xcode使用的编译器提供的，Xcode在你编译OC、C、C++程序时都会使用它。#import可保证头文件只被包含一次，无论此命令在该文件中出现了多少次。
    在C语言中，程序员通常时候基于#ifdef命令的方案来避免一个文件包含另一个文件而后者又包含前者的情况。而OC中，程序员使用#import命令来实现这个功能。

2. 什么是框架？
    框架是一种把头文件、库、图片、声音等内容聚集在一个独立单元中的集合体。
    Foundation框架处理的是用户界面之下的那些层的特性，比如数据结构和通信机制。
    每个框架都是一个重要的技术集合，通常包含数十个甚至上百个头文件。每个框架都有一个主头文件，包含了框架内所有的头文件。通过在主头文件中使用#import，就可以访问框架内所有功能。
    Xcode使用预编译头文件来加快读取速度。

3. fgets()从文件中读取一行文本，fgets()调用会保留每行之间用来断行的换行符，但我们不需要它，所以将换行符替换为\0，表示字符串的结束
    FILE *wordFile = fopen(“/word.txt”, “r”);
    char word[100];
    while (fgets(word, 100, wordFile)) {
        word[strlen(word) - 1] = ‘\0’;
        NSLog(“@s”, word);
    }

4. main()的参数
    通过main()的argc参数可以得知启动参数的数量。因为程序名总是用做启动参数而传递，所以argc的值至少是1，可能更大。如果用户不提供文件路径，那么argc的值将会是1。
    argv[1]保存着用户所提供的文件名。argv[0]保存着程序名

5. 什么是id？
   id是一种泛型，可以用来引用任何类型的对象。对象是一种包含代码的struct结构体，因此id实际上是一个指向结构体的指针。

6.Objective-C动态本质的副作用
  Objective-C不存在真正的私有方法，也无法把某个方法标识为私有方法，从而禁止其他代码调用它。

7. #import
   导入头文件有两种方法：使用引号和使用尖括号。带尖括号的语句用于导入系统文件，它是只读的；带引号的语句说明导入的是项目本地的头文件，可以编辑它。

8. @class
    OC引入了关键字@class来告诉编译器：“这是一个类，所以我只会通过指针来引用它”这样便一起就放心了：它不必知道关于这个类的更多信息，只要了解它是通过指针来引用的即可。
    @class创建了一个前向引用。这是在告诉编译器：“相信我。以后你自然会知道这个类到底是什么，倒是现在，你知道这些足矣”
    如果有循环依赖关系，@class也很有用。即A类使用B类，B类也使用A类。如果试图通过#import语句让这两个类相互引用，那么就会出现编译错误。但是如果在A.h文件中使用@class B，在B.h中使用@class A，那么这两个类就可以相互引用了。

9. 工厂方法
    如果你在声明方法时添加了加号，就是把这个方法定义为类方法。这个方法属于类对象，通常用于常见新的实例。我们称这种用来创建新对象的类方法为工厂方法。stringWithFormat: 就是一个工厂方法。

10. length和strlen()
    NSString的length方法能够精确无误地处理各种语言的字符串，如含有俄文、中文或者日文字符的字符串，以及使用Unicode国际字符标准的字符串。在C语言中处理这些国际字符串是件令人头疼的事情，因为一个字符占用的空间可能多于1个字节，这就意味着如strlen()之类只计算字节数的函数会返回错误的数值。

11. 字符串内是否还包含别的字符串
    hasPrefix: 检查字符串是否以另一个字符串开头
    hasSuffix: 检查字符串是否以另一个字符串结尾
    rangOfString: 检查字符串内的某处是否包含其他字符串

12. deleteCharactersInRange: 和rangeOfString: 一起使用删除字符串内的某段字符串

13. 切分数组
    使用NSArray *chucks = [string componentsSeparetedBySting: @“,”]; 可以将字符串string按其中的”,”切分成数组；
    使用string = [chucks componentsJoinedByString: @“-”]; 可以将数组chucks中的元素用”-”合并成字符串。

14. 装箱和开箱
    通常将一个基本类型的数据封装成对象的过程被称为装箱，从对象中提取基本类型的数据叫做开箱。OC不支持自动装箱功能。

15. NSNull
    使用NSNull可以表示空值
    [contact setObject: [NSNull null] forKey: @”home”];
    if ([contact objectForKey: @”home”] == [NSNull null]) {}

16. 每个对象都维护一个保留计数器。对象被创建时，其保留计数器的值为1；对象被保留时，其保留计数器的值加1；对象被释放时，其保留计数器的值减1；当保留计数器的值归0了，对象被销毁。在销毁对象时，首先调用对象的dealloc方法，然后收回其占用的内存以供其他对象使用。

17. 当对象接收到一条autorelease消息时，其保留计数器的值不会发生改变。该对象知识被放入NSAutoreleasePool当中。当自动释放池被销毁时，会向池中的所有对象发送release消息，所有被自动释放的对象都将其保留计数器的值减1.如果保留计数器的值归0了。则对象被销毁。在使用AppKit或UIKit的时候，自动释放池会在明确的时间创建或释放，比如在处理当前用户事件的时候。

18. Cocoa有三个关于对象及其保留计数器的规则：
    * 如果使用new、alloc或copy操作获得一个对象，则该对象的保留计数器的值为1；
    * 如果通过其他方法获得一个对象，则假设该对象的暴力计数器的值为1，而且已经被设置为自动释放；
    * 如果保留了某对象，则必须保持retain方法和release方法的使用次数相等。
通常ARC会在编译过程中通过插入这些语句来帮你执行保留或释放操作。

19. 在Cocoa中，分配和初始化时两个分离的操作：来自NSObjet的类方法alloc为对象分配一块内存区域并将其清零，实力方法init用于获得一个对象并使其运行。

20. 在你自己的初始化方法中，需要调用自己指定初始化函数或者超类的指定初始化函数，一定要将超类的初始化函数的返回值赋值给self，并返回你自己init方法的值。超类可能返回一个完全不同的对象。

21. 浅层复制、深层复制
    * 如果你使用的是浅层复制，那么不会复制所引用的对象，新复制的对象只会指向现有的引用对象。当复制一个NSArray类的对象时，复制的对象只会复制指向引用对象的指针，而不会复制引用对象本身。如果复制了一个包含了5个NSString对象的NSArray对象，那么你最终得到的是5个可供程序使用的字符串对象，而不是10个。这种情况下每个对象最终都会指向一个字符串。
    * 深层复制将复制所有的引用对象。如果NSArray的copy方法是深层复制，则在复制操作完成以后的到10个字符串对象。
